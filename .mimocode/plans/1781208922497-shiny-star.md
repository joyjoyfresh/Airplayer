# 音频播放 + 音视频同步 — 实现计划

## 现状分析

音频数据流已完整实现到解码器之前：
```
iOS → UDP(7022/7023) → AES-CBC解密 → RaopBuffer → IDecoder → PcmData → AirPlayReceiver
```

**缺失的 3 个环节：**
1. `NoOpDecoder` — 不做任何解码，输出空数据
2. `AirPlayReceiver.OnPCMData()` — 直接丢弃音频（注释：v1不处理音频）
3. App 层无音频播放组件

## 技术选型

| 组件 | 方案 | 理由 |
|------|------|------|
| AAC-ELD 解码 | **Vortice.MediaFoundation** MFT（已引用） | Windows 原生 MediaFoundation AAC 解码器，无需额外依赖 |
| PCM 播放 | **Windows.Media.Audio.AudioGraph** | WinUI3 原生 API，支持低延迟帧注入，精确时钟 |
| 音视频同步 | **音频 PTS 为主时钟** | AirPlay 标准做法，视频追赶音频 |

## 实现步骤

### 步骤 1：AAC-ELD 解码器
**新建文件：** `AirPlayer.Protocol/Decoders/AacDecoder.cs`

- 实现 `IDecoder` 接口
- 使用 `Vortice.MediaFoundation` 创建 MFT AAC 解码器（`MFT_CATEGORY_AUDIO_DECODER`）
- `Config()`: 初始化 MediaFoundation，创建 AAC decoder MFT，设置输入输出类型
- `DecodeFrame()`: 将压缩帧送入 MFT，拉取解码后的 PCM（16-bit, 44100Hz, 2ch）
- `GetOutputStreamLength()`: 返回一帧最大 PCM 大小（samples_per_frame * 2ch * 2bytes）
- 修改 `AudioListener.InitializeDecoder()`：根据 `session.AudioFormat` 选择 `AacDecoder` 或 `NoOpDecoder`

**修改文件：**
- `AirPlayer.Protocol/Listeners/AudioListener.cs` — `InitializeDecoder()` 根据格式创建正确解码器

### 步骤 2：暴露音频事件
**修改文件：**

`AirPlayer.Protocol/IAirPlayReceiver.cs`：
```csharp
event EventHandler<PcmData> OnPcmDataReceived;
event EventHandler OnAudioFlushReceived;
```

`AirPlayer.Protocol/AirPlayReceiver.cs`：
- 实现 `OnPCMData()`：触发 `OnPcmDataReceived` 事件
- 实现 `OnAudioFlush()`：触发 `OnAudioFlushReceived` 事件

### 步骤 3：音频播放器
**新建文件：** `AirPlayer.App/AudioSink.cs`

核心设计：
```
AudioGraph（系统音频时钟）
  └─ AudioFrameInputNode（帧注入节点）
       ← 从 AudioFrame 构造 PCM 数据帧
       ← 按 PTS 调度注入时机
```

- `Initialize()`: 创建 AudioGraph + AudioFrameInputNode
- `EnqueueFrame(PcmData)`: 将 PCM 数据包装为 AudioFrame，根据 PTS 计算注入时间
- `Flush()`: 清空缓冲队列
- `Dispose()`: 释放 AudioGraph

播放策略：
- 用 `ManualResetEventSlim` 或 `SemaphoreSlim` 控制节奏
- AudioFrameInputNode 的 `QuantumStarted` 回调中拉取下一帧
- 首帧到达时开始播放，积累 3-5 帧缓冲后再开始

### 步骤 4：MainWindow 集成
**修改文件：** `AirPlayer.App/MainWindow.xaml.cs`

- 订阅 `_receiver.OnPcmDataReceived` 和 `_receiver.OnAudioFlushReceived`
- 创建 `AudioSink` 实例（与 `_presenter` 生命周期一致）
- `Receiver_OnMirroringStarted`: 初始化 AudioSink
- `Receiver_OnMirroringStopped`: 释放 AudioSink
- 新增字段：`AudioSink? _audioSink`

### 步骤 5：音视频同步
**核心逻辑在 AudioSink 中实现：**

同步策略（音频为主时钟）：
1. AudioSink 记录首帧 PTS 作为 `basePts`
2. 每帧计算 `playbackTime = pts - basePts`（相对时间）
3. AudioFrameInputNode 的 `QuantumStarted` 事件驱动帧消费
4. 视频端（VideoPresenter）后续可通过读取 AudioSink 的时钟来对齐（本期先实现音频独立播放，同步后续迭代）
