# 投屏音频修复说明（无声 + 音画同步）

## 根本原因（已用日志逐条证实）

1. **解码器产不出任何 PCM（头号原因）。** 投屏音频是 **AAC-ELD**。
   - 微软 Media Foundation 的 AAC 解码器不支持 ELD（且你机器上未注册，`REGDB_E_CLASSNOTREG`）。
   - 我一度改用 FFmpeg(avcodec)，它能加载、能初始化（日志 `配置成功`），但**每帧
     `avcodec_receive_frame` 返回 `AVERROR_BUG`**，输出全静音。我用相同的调用方式解 AAC-LC
     完全正常，证明是 **FFmpeg 原生解码器解不了 ELD**（这也是 RPiPlay/UxPlay 都用 fdk-aac 的原因）。
   - 结论：AAC-ELD 必须用 **fdk-aac**。
2. **waveOut 回调常量写错**（`0x3C00` 应为 `0x3BD`），第一块缓冲播完即停。已修。
3. **解码帧长度取错**（用固定 4096 而非实际字节数），会噪音/不同步。已修。

## 现在的方案与代码改动

- AAC-ELD 解码器：`Decoders\FdkAacEldDecoder.cs`（fdk-aac，标准 ASC `F8 E8 50 00`）。
- `AudioListener`：ELD 固定走 fdk-aac，并修正按实际解码字节数投递 PCM。
- 重写 `AudioSink`：修正 `WOM_DONE=0x3BD`、事件+工作线程避免回调死锁、
  有界低延迟队列（~150ms 丢旧）保证与视频对齐 → 音画同步。
- 全链路清晰日志：`[FDK]`、`[DEC]`、`[AUDIO-C/D]`、`[PLAY]`，缺库/错误都打横幅。
- （FFmpeg 那条路已废弃，`FfmpegAacEldDecoder.cs` 置空保留结论。）

## 唯一还需要的东西：fdk-aac.dll（约 1.5MB，x64）

我无法替你打进仓库（构建环境拿不到它）。三种获取方式，**任选其一即可**：

**方式 0：程序自动下载（推荐，零操作）**
重新编译运行后，若检测到缺库，程序会自动从 MSYS2 官方仓库下载 `fdk-aac.dll` 放到输出目录。
日志会出现：`[FDK] 未发现 fdk-aac.dll，尝试自动从 MSYS2 下载...` → `[FDK] 自动下载成功`。
（需要这台机器能联外网；公司内网/代理可能拦截，则用下面方式。）

**方式 1：一键脚本**
```
powershell -ExecutionPolicy Bypass -File tools\get-fdk-aac.ps1
```

**方式 2：vcpkg（最稳，但慢）**
```
git clone https://github.com/microsoft/vcpkg
.\vcpkg\bootstrap-vcpkg.bat
.\vcpkg\vcpkg install fdk-aac:x64-windows
copy .\vcpkg\installed\x64-windows\bin\fdk-aac.dll AirPlayer.App\native\
```

## 操作步骤

1. **VS2022：生成 → 清理解决方案 → 重新生成解决方案**（务必重新生成，旧 exe 不含新代码）。
2. 运行，投屏。
3. 看 `AirPlayer.Protocol\airplay-video.log`，成功标志：
   ```
   [FDK] 找到原生库: ...\fdk-aac.dll        （或：自动下载成功）
   [FDK] AAC-ELD 解码器就绪
   [FDK] 解码OK 累计成功=... out=1920
   ```
   `AirPlayer.App\audio-debug.log` 会出现 `[ENQ]/[FILL]/[QUEUE]` 持续滚动、`[PLAY] 健康度`。

   若仍无声，日志会**直接写明原因**：
   - `[FDK][致命] 未找到 ... fdk-aac.dll` 且自动下载失败 → 用方式 1/2 手动放一个。
   - 仍是 `[BUF-DECODE] ERROR` 且无 `[FDK]` 行 → 跑的还是旧 exe，没重新生成。

## 备注
- `AirPlayer.App\native\` 里若有 avcodec-*/libx264/libx265 等 FFmpeg DLL，是早期尝试遗留，
  可以删掉（构建已不再拷贝它们，只拷 `*fdk*.dll`）。
- `native\` 可加进 `.gitignore`。
